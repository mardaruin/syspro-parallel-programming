
    Could main thread observe x == 1, r_y == 0, r_z == 0?
    Could main thread observe x == 2, r_y == 0, r_z == 1?
    Could main thread observe x == 1, r_y == 0, r_z == 1?

Answer:
    1. Да, для цепочки A1-A2-A3-B1-B2-B3-A4-B4-A5-B5-A6-B6 выполняется x == 1, r_y == 0, r_z == 0:
        так как в a_x b_x присваиваются 0, оба потока пройдут по условию, присвоят в y z нули, далее оба запишут
        x=1, а потом в r_y r_z нули

    2. Да, для цепочки A1-A2-B1-B2-A3-A4-A5-A6-B3-B4-B5-B6 выполняется x == 2, r_y == 0, r_z == 1:
        A4: y=0   A5: x=1   A6: r_y=0, x=1
        B4: z=1   B5: x=2   B6: r_z=1, x=2

    3. Заметим, что, если поток не проходит по условию, то он не меняет x, а в r_y/r_z соответственно записывает -1.
        Таким образом, для выполнения условия x == 1, r_y == 0, r_z == 1 необходимо, чтобы оба потока в a_x b_x присвоили нули.
        При этом несложно заметить, что, так как относительно потоков y и z - независимые переменные, каждый поток после
        исполнения записывает в r_y r_z то значение, которое было в x на момент A4/B4. То есть нам надо, чтобы на момент A4 x==0,
        B4 x==1.таким образом A5 должно выполниться раньше B4.
        Но также, чтобы x==1 после исполнения необходимо, чтобы B4 исполнилось раньше A5 - иначе x==2. Противоречие
       Итого: ситуация x == 1, r_y == 0, r_z == 1 невозможна.
