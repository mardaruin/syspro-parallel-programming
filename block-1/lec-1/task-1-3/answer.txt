Заметим, что, если при выполнении C1 y != 2, то сам поток C ничего не меняет. А чтобы при выполнении C1 y == 2
    необходимо, чтобы при A3 a_x == a_z == 1 (x z нельзя увеличить более, чем на 1 после всех потоков), а для этого
    B4 должно исполниться раньше A1 (условие 0).
Итого: если первые команды не соответствуют условию 0, поток C можно даже не рассматривать, он ничего не меняет

0. Условие 0 выполнено: B1-B2-B3-B4-A1-A2-A3-C1-C2-C3-C4:
    После B: x==z==1, y==0
    После A: y==2, x==z==1
    После C: x==0, y==2, z==1

Далее условие 0 не будет выполнено, поэтому поток C не будет в цепочке исполнения, его операции
    можно ставить в любом месте, это ни на что не повлияет.
Также заметим, что x z без выполнения условия 0 может изменять только B, более того, без выполнения условия 0
    по окончанию потоков они всегда будут равны 1. Поэтому без условия 0 возможны только 3 разных исхода:
    y==0, y==1, y==2 (x==z==1 во всех трех):

1. A1-A2-A3-B1-B2-B3-B4:
    После A: x==y==z==0
    В конце: x==1, y==0, z==1

2. A1-B1-B2-B3-B4-A2-A3:
    A в a_x присвоит 0, далее B изменит z=1, A в a_z присвоит 1, а потом сложит и присвоит в y:
    x==1, y==1, z==1

3. B1-B2-B3-B4-A1-A2-A3:
    B увеличит x z на 1, A сложит их и положит в y:
    x==1, y==2, z==1
